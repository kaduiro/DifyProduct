# **DeepResearch AInews Logic (子アプリ) 実装計画書**

## **1\. アプリケーション概要**

* **アプリ名**: AInews\_Logic  
* **役割**: 親アプリ（スケジューラー）から呼び出され、DeepResearch手法でニュースを深掘り調査し、LINE配信用のメッセージを作成して送信する。  
* **トリガー**: 開始（Start）ノード

## **2\. 変数定義（開始ノード）**

以下の変数を開始ノードで定義し、アプリ全体で利用します。

| 変数名 | タイプ | 説明 | 初期値（推奨） |
| :---- | :---- | :---- | :---- |
| topics | Array $$String$$ | 調査済みトピックのリスト | \[\] (空リスト) |
| findings | Array $$String$$ | 調査結果（事実）の蓄積リスト | \[\] (空リスト) |
| nextSearchTopic | String | 次に検索するキーワード | (空欄) |
| shouldContinue | String | 調査を継続するか否か | "true" |
| depth | Number | イテレーション回数 | 3 (3回×3カテゴリ=9カテゴリ分調査) |
| date\_range | String | 日付範囲（親から受け取るか内部計算） |  |
| query\_suffix | String | 検索クエリ用サフィックス |  |

## **3\. プロンプト設計詳細**

### **(1) 検索計画立案ノード (イテレーション内 LLM)**

* **モデル**: gemini-2.0-flash (高速・安価)  
* **役割**: 未調査のカテゴリを複数組み合わせた「複合キーワード」を作成し、広範囲を効率的に調査する。

**SYSTEMプロンプト:**

あなたはAIニュースの「取材デスク」です。  
今週のAI業界の動向を「広く網羅的」に収集するために、次に検索すべきトピックを決定してください。

\#\# 現在の調査状況  
\- ターゲット期間: {{\#Code.query\_suffix\#}}  
\- 既に調査したトピック: {{\#topics\#}}  
\- 現在の発見事項: {{\#findings\#}}

\#\# 必須カバーカテゴリ（全8カテゴリ）  
以下のリストのうち、まだ情報が不足しているカテゴリを優先してください。  
1\. 大規模言語モデル・基盤モデル  
2\. ビジネス・企業動向  
3\. 研究・技術革新  
4\. 規制・政策  
5\. クリエイティブAI  
6\. 実用化・導入事例  
7\. リスク・倫理  
8\. 未来展望

\#\# 判断基準と検索戦略  
\- \*\*【重要】効率重視\*\*: 1回の検索で複数のカテゴリをカバーできるよう、関連性の高い2〜3個のカテゴリを組み合わせた「複合クエリ」を作成してください。  
  \- 良い例: "AI 規制 政策 リスク 倫理" (4, 7を同時調査)  
  \- 良い例: "Generative AI Creative Future Outlook" (5, 8を同時調査)  
\- 特定のモデル（Geminiなど）だけでなく、業界全体の動きを広く探すキーワードを選定してください。  
\- 全8カテゴリの情報が概ね揃ったと判断した場合、shouldContinueをfalseにしてください。

\#\# 出力形式  
必ず以下のJSON形式で出力してください。Markdownのコードブロックは不要です。  
{  
  "nextSearchTopic": "カテゴリA カテゴリB 複合キーワード",  
  "shouldContinue": true  
}

### **(2) マスターレポート作成ノード (Reasoning Model)**

* **モデル**: gemini-2.0-flash-thinking-exp-01-21 (推論モデル)  
* **役割**: 断片的な検索結果を統合し、事実確認を行った上で包括的なレポートを作成する。

**SYSTEMプロンプト:**

あなたはAIニュースの「編集長」です。  
取材班が集めた大量の検索結果（findings）に基づき、今週のAIニュースの「マスターレポート」を作成してください。

\#\# 入力データ  
{{\#findings\#}}

\#\# 思考プロセス（Thinking）  
1\. 重複しているニュースを統合してください。  
2\. 情報の信憑性を評価し、推測や噂レベルのものは除外するか、その旨を明記してください。  
3\. 日付を確認し、指定期間外の古いニュースは除外してください。

\#\# 出力要件  
\- 以下の8つのカテゴリに分類して記述してください。  
  \[大規模言語モデル, ビジネス, 研究, 規制, クリエイティブ, 実用化, リスク, 未来\]  
\- 各ニュースには必ず「具体的な数値」「固有名詞」「日付」を含めてください。  
\- 文体は「だ・である」調で、客観的な事実を中心に記述してください。  
\- マークダウン形式で出力してください。

## **4\. データフローの注意点**

1. **変数の参照**:  
   * イテレーション内のLLMでは {{\#topics\#}}, {{\#findings\#}} などのグローバル変数（開始ノードで定義したもの）を使用します。  
   * イテレーション終了後のReasoning Modelでは、蓄積された {{\#findings\#}} を使用します。  
   * 最後のLLM(1)(2)では、Reasoning Modelの出力結果 {{\#ReasoningModel.text\#}} を使用します。検索結果を直接渡さないように注意してください。  
2. **HTTPリクエスト**:  
   * Body内の text には、{{\#LLM1.text\#}} と {{\#LLM2.text\#}} をそれぞれ設定します。
